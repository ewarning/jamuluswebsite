Servers - openbaar

Waarom wordt mijn server niet weergegeven in de lijst? Waarom registreert hij zich niet?

Als je correct geregistreerd bent (dit kun je hier zien) en jij of je vrienden kunnen je server niet zien, dan kan het zijn dat je moet wachten of je client moet starten met de --showallservers optie en vanaf daar verbinding proberen te maken (zie opdrachtregelopties hoe je een client start met een configuratie optie).

Als je een bericht ziet dat de server vol is, raadplaag dan deze opmerking over adresboek servers.

Je kunt controleren of de server in het relevante genre wordt vermeld door deze hier te controleren.

Servers - Prive

Ik gebruik mijn client op dezelfde machine/hetzelfde netwerk als mijn server, maar ik kan er geen verbinding mee maken

Verbind je client met 127.0.0.1 (of localhost als dat werkt). Als je de client op een andere machine dan de server uitvoert maar op hetzelfde netwerk, maak dan verbinding met het lokale netwerk adres van de server. Maak geen verbinding via het openbare (WAN) adres van de server.

Welk adres geef ik aan mensen zodat ze verbinding kunnen maken met mijn server?

Dit moet je openbare IP-adres zijn (zoek dat op bij Google). Verbind je eigen Jamulus-client met het lokale IP-adres van de server (127.0.0.1 als deze zich op dezelfde machine als je client bevindt). Houd er rekening mee dat je openbare IP-adres kan veranderen - zie de opmerking over “dynamische DNS” in deze handleiding.

Welke poort nummers kan ik gebruiken?

Je kunt de server zo instellen dat deze op een aangepaste poort luistert met de optie --port'. Als je dit doet, moet je mensen vertellen op welke poort ze verbinding moeten maken. Ze moeten het poortnummer toevoegen aan het adres van je server in het formaat [serveradres]:[poortnummer]`.

Is een prive server een directe server zonder contact met de Jamulus adresboek server?

Ja. Zie servertypen

Kan ik mijn prive server omzetten in een openbare?

Ja. Vink gewoon het vakje “Maak mijn server openbaar” in het servervenster aan en vul de details in. Als je de server registreert, gebruik dan geen port forwarding aangezien dit kan voorkomen dat anderen je server gebruiken.

Headless Linux: Ik heb een server gemaakt, maar deze registreert zich bij de adresboek server en dat wil ik niet.

Gebruik de optie --directoryserver (-e) niet bij het starten van je server om een prive server te maken.

Niemand kan verbinding maken met mijn server - maar ik kan lokaal verbinding maken

Sommige ISP’s gebruiken technieken zoals Address plus Port (A+P) of Carrier-grade NAT (CGN) om adresruimte te besparen. Dit maakt het onmogelijk om services zoals Jamulus thuis te hosten (en kan betekenen dat je ook geen serverlijsten in je client kunt zien, of dat je een aantal specifieke servers mist).

Om te detecteren of CGN het probleem is, gaat je naar het configuratiescherm van jouw router (meestal 192.168.X.X) en kijk je naar de WAN-statuspagina. Als je vermelde IPv4-adres niet overeenkomt met het adres dat je hier ziet, betekent dit dat CGN mogelijk is ingeschakeld. Of als je problemen ondervindt, zoals het niet kunnen verbinden met servers of een lege serverlijst, dan kan dit ook een aanwijzing zijn dat je ISP het probleem is.

Om het probleem op te lossen: als je een openbare server gebruikt, probeer dan port forwarding. Als je een prive server gebruikt en al port forwarding uitvoert, neem dan contact op met je ISP-ondersteuning en vraag hen om je uit CGN te verwijderen en je een echt WAN IP-adres toe te wijzen omdat je een server wilt hosten. Als CGN niet het probleem is, moet je mogelijk je ISP vragen om je een WAN IP-adres te geven (of een vast adres om het gebruik van DDNS te vermijden, indien mogelijk). Als al het andere faalt, overweeg dan om op een cloudplatform te hosten (sommige providers hebben gratis proefperiodes).
